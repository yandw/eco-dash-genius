## 目标
在 `草稿区`（`src/pages/ent/EntDraft.tsx`）的 Tabs 中，紧挨现有的「目标分解流程图」后，新增一个 Tab：「**能耗考核流程图**」，用 SVG 泳道图呈现两类企业（区管 / 市管）从企业侧填报到政府侧三类账号审核的完整流程。

## 涉及文件
- `src/pages/ent/EntDraft.tsx`（唯一改动文件，纯展示，无业务逻辑）

## Tab 结构
```
Tabs
├─ 目标分解流程图（已有，不动）
└─ 能耗考核流程图（新增）
   ├─ 子图 A：类型A — 区下属单位（区管企业）
   └─ 子图 B：类型B — "百家"、"千家"、通信业企业（市管企业）
```
两张子图各自独立 SVG 泳道，纵向排列，复用现有 SVG 风格（marker arrow / 泳道背景 / 颜色 token），保证视觉与已有图一致。

## 子图 A — 区管企业（类型A）泳道结构
两条泳道：

```text
企业侧（区管企业）   │  [年报自动提取数值]  ──系统自动汇总──▶
                    │  （灰色虚框，标注"企业无需填写"）
────────────────────┼──────────────────────────────────────
政府侧              │  ┌─区负责人──┐  ┌─市级中心对口人─┐  ┌─市级管理员─┐
                    │  │修改实际值 │  │ 修改实际值     │  │修改实际值  │
                    │  │+ 考核结果 │→│ + 考核结果     │→│+ 考核结果  │→ 结果同步企业侧
                    │  │ 提交      │  │ 提交           │  │ 提交       │
                    │  └──────────┘  └────────────────┘  └────────────┘
```
- 三类政府账号横向并列，箭头表示均可修改并提交（不强制顺序，用并列+说明文字"任一角色提交即同步"）。
- 末端用一根回流虚线箭头到企业侧，标注"结果同步显示"。

## 子图 B — 市管企业（类型B）泳道结构
三条泳道：

```text
企业侧（市管企业）    │ ① 填写自评分 / 评分依据 / 上传证明材料 ──▶ ② 点击提交
─────────────────────┼─────────────────────────────────────────────────
政府侧·市级中心对口人 │ 收到提交 ─┬─ 退回重填 ──回流──▶ 企业侧重新填写
                     │           └─ 填写考评分+评分依据 ──保存──▶ 同步企业
─────────────────────┼─────────────────────────────────────────────────
政府侧·市级管理员     │ 收到提交 ─┬─ 退回重填 ──回流──▶ 企业侧重新填写
                     │           └─ 填写考评分+评分依据 ──保存──▶ 同步企业
```
- 「退回重填」用橙/警示色虚线箭头从政府侧指回企业侧"填写"节点。
- 「保存 → 同步企业」用主色虚线箭头到企业侧最右侧的"考评结果展示"小卡。
- 两个政府角色泳道结构对称，标注"任一角色操作均生效"。

## 视觉规范
- 复用现有图的：`hsl(var(--primary))` / `hsl(var(--success))` / `hsl(var(--muted))` / `hsl(var(--border))` / `hsl(var(--foreground))` token；箭头 marker `#arrow`（每个 SVG 内独立 defs 一份，避免 ID 冲突，如 `arrow-assess-a`、`arrow-assess-b`）。
- 退回箭头使用 `hsl(var(--destructive))` 或 `hsl(var(--warning))` 区分。

- 子图下方各放一个三/两列说明卡片网格，简述每个角色的操作权限（与现有"目标分解"图底部说明一致的样式）。
