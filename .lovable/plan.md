## 岗位备案模块重塑

参照原平台的 3 个 Tab 结构（基本信息 / 能源管理岗位备案 / 碳排放管理岗位备案），重新设计企业侧填报和政府侧查看的界面。岗位备案 **不需要审核**——企业填写后政府即可查看。

### 业务模型

岗位备案分为 3 个 Tab：
1. **基本信息** — 企业基础资料（统一信用代码、行业、园区、统计员等），与企业设置共享
2. **能源管理岗位备案** — 主管领导信息 + 负责人信息 + 备案文件 + 管理人员名册
3. **碳排放管理岗位备案** — 同上结构（独立一套数据）

### 企业侧 `/ent/posts`（重塑后）

**顶部摘要带**：
- 企业卡片（名称 + 信用代码）
- 备案完成度（基本信息 / 能源岗位 / 碳排岗位 三个状态徽章）
- 最近更新时间

**Tab 区**（三个 Tab，图标化）：

**Tab 1 · 基本信息**
- 双列字段网格：信用代码、企业名称、行业分类、工业园区、能耗级别、是否万家、统计员姓名/电话、注册资本、注册日期、法人代表、传真、邮编、所在区县、地址、所属区县、所属集团
- 顶部「编辑」按钮切换只读 / 编辑态
- 字段分组分隔（基础资料 / 联系方式 / 注册信息）

**Tab 2 · 能源管理岗位备案**
- **主管领导信息**卡片：姓名、职务、职称、电话、邮箱
- **负责人信息**卡片：姓名、部门、职务、职称、专业、文化程度、是否取得能源管理岗位证书、获证日期、证书证号、电话、邮箱、备案日期、工作经验（多行）
- **备案文件**：拖拽上传区 + 已上传文件列表（下载/删除）
- **管理人员**子表：姓名、岗位分工、受聘时间、技术职称、是否取得证书、证书证号、联系电话；行内编辑、新增、删除、搜索
- 操作区：导出 / 上传备案表 / 保存

**Tab 3 · 碳排放管理岗位备案**
- 与 Tab 2 完全相同的结构（独立数据集）
- 字段「是否取得能源管理岗位证书」改为「是否取得碳排放管理岗位证书」

**填报体验改进**：
- 折叠分组（领导/负责人/文件/人员），减少视觉负担
- 必填项红星标记 + 字段级保存提示
- 「同步企业设置」按钮——从企业资料一键拉取主管领导
- 表单顶部进度条显示完成度
- 自动保存草稿（提示「已自动保存」）

### 政府侧 `/gov/posts`（重塑后，无审核）

**列表页（默认视图）**：
- 顶部 KPI：纳入企业总数 / 已完成备案 / 部分备案 / 未备案 / 证书将到期
- 筛选器：区县 / 行业 / 备案状态 / 关键字
- 企业列表表格：企业名称、所属区县、行业、能源岗位状态、碳排岗位状态、管理人员数、最近更新、操作（查看详情）
- 状态徽章：✓ 已备案 / ⚠ 待完善 / ○ 未备案

**详情页 `/gov/posts/:entId`**：
- 与企业侧 Tab 结构完全一致，但**全部只读**
- 顶部企业基本信息条
- 三个 Tab 内容只读展示
- 右上角「导出 PDF」「导出名册」按钮
- 备案文件可在线预览/下载
- 管理人员表格只读 + 排序/搜索

- `src/components/posts/PostStaffTable.tsx` — 管理人员子表
- `src/components/posts/FileUploadList.tsx` — 备案文件上传/列表
- `src/pages/gov/GovPostDetail.tsx` — 政府侧企业详情页

**重写文件**：
- `src/pages/ent/EntPosts.tsx` — 改为 3-Tab 填报界面
- `src/pages/gov/Posts.tsx` — 改为企业列表 + KPI

**路由更新**（`src/App.tsx`）：

**视觉风格**：沿用现有"深海蓝 + 亮蓝"色系，复用 `panel`、`bg-gradient-primary`、状态徽章配色（success/warning/destructive），与档案管理模块视觉一致。
