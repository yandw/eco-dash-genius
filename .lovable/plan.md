## 目标

在企业侧新增一级菜单"草稿区"，页面内展示一张"双控考核 - 目标分解"业务流程的泳道图（用于开发评审）。

## 实施步骤

### 1. 新增路由与菜单项
- 在 `src/components/AppSidebar.tsx` 的 `entItems` 中，新增一项一级菜单"草稿区"（图标用 `FileEdit` 或 `PencilRuler`），路由 `/ent/draft`。
- 在 `src/App.tsx` 中注册路由 `/ent/draft` → 新页面 `EntDraft`。

### 2. 新建草稿区页面
- 新建 `src/pages/ent/EntDraft.tsx`。
- 页面顶部使用与其它企业侧页面一致的标题样式（标题"草稿区"，副标题"业务流程草图与评审用图"）。
- 主体放置一张卡片，标题"双控考核 - 目标分解 业务流程图"，内嵌泳道图。

### 3. 泳道图实现方式
使用纯 SVG + Tailwind 语义 token 手绘泳道图（不引入 mermaid/第三方图表库），优点：
- 完全符合现有设计系统配色（primary / success / muted 等）
- 评审清晰、可直接缩放、无外部依赖

布局采用三条横向泳道：
```text
┌──────────────────┬──────────────────────────────────────────────────────┐
│ 政府侧           │ [市级管理员] 创建任务                                 │
│ (市级管理员)     │   ├─ 区下属单位碳排放目标分解任务                      │
│                  │   └─ "百家"/"千家"/通信业企业碳排放目标分解任务        │
├──────────────────┼──────────────────────────────────────────────────────┤
│ 企业侧           │      ↓ 任务下发                                       │
│ (填报企业)       │ [企业] 填报碳排放目标值 → 提交                         │
├──────────────────┼──────────────────────────────────────────────────────┤
│ 政府侧           │      ↓ 数据汇总                                       │
│ (区级 / 中心     │ ┌─区级管理员─┐  ┌─市级中心对口人─┐  ┌─市级管理员─┐    │
│  对口人 /        │ │ 本区企业   │  │ 对口企业       │  │ 全市企业    │    │
│  市级管理员)     │ │ 可查看/修改│  │ 可查看/修改    │  │ 可查看/修改 │    │
│                  │ └────────────┘  └────────────────┘  └─────────────┘    │
└──────────────────┴──────────────────────────────────────────────────────┘
```

图元规范：
- 泳道分隔：水平线 + 左侧角色标签列（宽 ~140px，背景 `bg-muted/40`）
- 节点矩形：圆角 8px，1px 边框，分类配色
  - 政府侧创建/汇总节点：`border-primary` + `bg-primary/5`
  - 企业侧填报节点：`border-success` + `bg-success/5`
  - 决策/动作箭头：`stroke-foreground/60`，箭头使用 `<marker>` 定义
- 文字：节点标题 14px 粗体，描述 12px `text-muted-foreground`
- 整图最小高度 ~520px，宽度自适应（max-w 完整内容宽度）

### 4. 视觉与一致性
- 卡片包裹使用项目已有 `Card` 组件
- 颜色全部走语义 token（不写死 hex）
- 在图下方加一段简短文字说明角色权限（区级 / 市级中心对口人 / 市级管理员的修改范围），便于评审
