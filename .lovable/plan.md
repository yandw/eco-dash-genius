## 任务管理 ↔ 目标分解 / 双控考核 联动方案

### 1. 任务管理列表移除"已归档"
- 文件：`src/pages/gov/AssessTasks.tsx`
- 渲染前过滤掉 `status === "已归档"` 的任务，列表只显示 未开始 / 进行中 / 已结束。
- 底层 `AssessTaskStatus` 类型保留 4 项（兼容历史 seed），但列表与年份下拉的可选年份均基于"非已归档"任务计算。
- 新建/编辑弹窗 `AssessTaskFormDialog` 已不含"已归档"，无需改动。

### 2. 任务类型 ↔ 业务界面映射
| 任务类型 | 关联界面 | 子 Tab（市级） |
|---|---|---|
| 区下属单位碳排放目标分解 | 目标分解 | 区下属单位碳排放目标分解 |
| "百家"、"千家"、通信业企业碳排放目标分解 | 目标分解 | 百千通碳排放目标分解 |
| 区下属单位能耗考核 | 双控考核 | 区下属单位能耗目标考核 |
| "百家"、"千家"、通信业企业能耗考核 | 双控考核 | 百千通能耗考核 |

类型分组：
- "目标分解类" = 上表前 2 项
- "双控考核类" = 上表后 2 项

### 3. 共享 helper（`src/mocks/assessTasks.ts`）
新增前端工具：
- `listActiveTasks()`：返回非已归档任务。
- `hasActiveTask(year, types: AssessTaskType[])`：该年内是否存在指定类型的任务。
- `hasAnyActiveTask(types: AssessTaskType[])`：跨年度是否存在指定类型任务。
- `listActiveYears(types)`：存在指定类型任务的年份并集，降序。

### 4. 通用占位组件 `AssessEmptyState`
- 新增文件：`src/components/assess/AssessEmptyState.tsx`
- 内容：居中卡片 + `ClipboardList` 灰色大图标 + 主文案 + 副文案 + 可选「前往任务管理」按钮（仅市级管理员显示，跳转 `/gov/assess/tasks`）。
- 在政府/企业、市级/区级、目标分解/双控考核 各空状态中统一复用，文案不同。

### 5. 政府侧 · 目标分解（`src/pages/gov/AssessGoal.tsx`）
市级与区级视图均接入：
- **市级**：
  - 顶部不再硬编码 `YEARS`，改为 `listActiveYears(目标分解类)`；
  - 若整体无任何"目标分解类"任务 → 整页只显示占位图：「请在任务管理中创建目标分解任务」（含跳转按钮）；
  - 若该年只缺其中一个 Tab 类型，则隐藏对应 Tab；
  - 若该年两类都没任务但其他年份有 → 切到首个有任务的年份。
- **区级**：
  - 检查"区下属单位碳排放目标分解"是否存在「当年（默认本期年）」任务；不存在 → 占位「今年目标分解未开始」（无跳转按钮，因为区级无任务管理权限）。

### 6. 政府侧 · 双控考核（`src/pages/gov/AssessDual.tsx`）
- **市级**：
  - 年度按钮基于 `listActiveYears(双控考核类)`；
  - 无任何双控类任务 → 占位「请在任务管理中创建考核任务」（市级管理员含跳转按钮）；
  - Tab 按该年任务存在性动态显示/隐藏。
- **区级**：
  - 检查"区下属单位能耗考核"在当年是否有任务；无 → 占位「今年考核未开始」。

### 7. 企业侧 · 目标分解（`src/pages/ent/EntAssessGoal.tsx`）
- 年度按钮基于"目标分解类"有效年份；
- `scope = district` 对应「区下属单位碳排放目标分解」、`scope = city` 对应「百千通碳排放目标分解」；scope Tab 按当年存在性动态显示；
- 当年无任何目标分解任务 → 占位「今年目标分解未开始」（无跳转按钮）。

### 8. 企业侧 · 双控考核（`src/pages/ent/EntAssessDual.tsx`）
- 由 `entType` 决定关联类型：区下属单位 → 区下属能耗考核；百/千/通信业 → 百千通能耗考核；
- 年度按钮基于该类型有效年份；
- 当年无对应考核任务 → 占位「今年考核未开始」。

### 9. 不改动范围
- 不修改任务详情、企业名单弹窗、任务模板。
- 不修改"区/集团管理"。
- 不引入后端联动；通过 `useSyncExternalStore` 自动响应任务管理增删改。
