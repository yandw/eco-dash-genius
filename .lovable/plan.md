## 文档大纲

### 一、背景

- 业务背景：上海市对工业和通信业重点用能单位实行"目标分解 + 双控考核"年度闭环管理。原线下 Excel 模式存在数据口径不一、修改追溯难、附件丢失等问题。
- 业务目标：将"任务下发 → 企业填报/自评 → 区级审核 → 市级汇总 → 盖章证明 → 归档"全流程线上化；指标自动测算与达标自动判定；中心调整有痕。
- 角色：市级管理员（节能中心）、区级管理员（区经信委）、区管企业、市管企业（百千通信业）。

### 二、需求描述

#### 2.1 功能概述

考核管理模块包含 2 大业务（目标分解 + 双控考核）+ 2 项基础配置（任务管理、区/集团管理），共 9 个页面：

| 类别 | 页面 | 路由 | 角色 |
|---|---|---|---|
| 基础配置 | 任务管理 | /gov/assess/tasks | 市级 |
| 基础配置 | 区/集团管理 | /gov/assess/orgs | 市级 |
| 政府-目标分解 | 目标分解（市级/区级） | /gov/assess/goal | 市级 + 区级 |
| 政府-目标分解 | 区下属企业目标分解详情 | /gov/assess/goal/district/:districtId | 市级 |
| 政府-双控考核 | 双控考核（市级/区级） | /gov/assess/dual | 市级 + 区级 |
| 政府-双控考核 | 区下属企业能耗考核详情 | /gov/assess/dual/district/:districtId | 市级 |
| 政府-双控考核 | 百千通信业企业评分详情 | /gov/assess/dual/bq/:entId | 市级 |
| 企业-目标分解 | 重点单位碳排放双控目标分解 | /ent/assess/goal | 企业 |
| 企业-双控考核 | 重点单位能耗双控考核结果 | /ent/assess/dual | 企业 |

#### 2.2 需求目标

1. 任务驱动：所有考核内容由市级在「任务管理」中创建任务后才出现入口，实现"无任务无操作"。
2. 流程可控：四级流转 企业填报 → 区级审核/盖章 → 市级汇总 / 退回，状态留痕、可回滚。
3. 数据一致：实际能耗、扣除绿电后能耗、达标判定均由系统自动取数与计算，避免人工口径偏差。
4. 修改有痕：中心负责人对企业目标的调整字段高亮、可查阅修改记录（字段/旧值/新值/操作人/时间/备注）。
5. 附件管理：盖章证明、考评报告、自评附件统一上传/预览/下载/删除。

#### 2.3 需求范围

包含：
- 任务管理（4 类任务的 CRUD、企业名单导入、模板下载、企业名单查看分页）。
- 区/集团管理（区管理单位、企业集团 双 Tab CRUD，含中心对口人配置）。
- 目标分解：市级双 Tab（区下属/百千通信业）、区级单视图（本区下属企业）、企业端双类型（区管/市管）填报。
- 双控考核：市级双 Tab（区下属/百千通信业）、区级单视图（本区企业评分）、市级对单家百千企业评分详情、企业端结果查看与百千企业自评。
- 状态机：任务（未开始/进行中/已结束/已归档）、区级考核（待考核/考核中/完成考核）、企业填报（草稿/已提交/中心已调整）、百千企业（待提交/已提交/考核中/已完成）。
- 通用能力：年度切换（仅显示有任务的年份）、倒计时徽标、空态、盖章证明上传/预览/删除、Excel 导出。

不包含：
- 系统管理与账号体系（独立 PRD）。
- 节能档案、月报、年报、能源审计（独立 PRD）。
- 离线公文流转（盖章环节为线下，平台仅承载已盖章 PDF）。

#### 2.4 详细需求描述

按页面拆解：页面说明 → 区块 → 字段表 → 交互/状态 → 异常/边界 → 权限。

**2.4.1 任务管理 /gov/assess/tasks**（市级）
- 列表字段：序号、年份、类型（4 类，碳排=绿色徽标 / 能耗=蓝色徽标）、开始时间、结束时间、状态、创建日期、企业名单（链接 → 弹窗）、操作（编辑、删除）。
- 筛选：年份（动态）、类型；分页（默认 10/页）。
- 新建/编辑表单：年份（YearPicker，必填）、类型（必选 4 选 1）、开始时间（必填）、结束时间（必填，≥ 开始）、状态、企业名单（.xlsx/.xls/.csv，必填，含模板下载 — 4 种类型对应 4 套表头）。
- 企业名单弹窗：序号、区名称（仅"百千通信业能耗考核"不含此列）、统一信用代码、企业名称；分页。
- 状态机：未开始 → 进行中 → 已结束 → 已归档（已归档不再出现在企业/区级视图）。
- 权限：仅 isCityAdmin = true 可见；非市级显示「无权限访问」。
- 异常：结束 < 开始 → toast 报错；未上传名单 → toast 报错；删除前 AlertDialog 确认。

**2.4.2 区/集团管理 /gov/assess/orgs**（市级）
- 双 Tab：区管理单位 / 企业集团；列字段：序号、区名称(集团)、单位全称、单位负责人 + 联系方式、地址、中心对口人、状态（正常/关闭，徽标）、操作。
- 表单字段：区名称/集团（必填）、单位类型（区/集团）、状态、单位全称（必填，全表去重）、单位负责人（必填）、联系方式（7-11 位数字）、联系地址、中心对口人（下拉，预置 6 人：李雯/陈玲凯/蒋伊莹/刘鑫/贾连海/李婕）。
- 异常：单位全称重复 → 表单错误"该单位已存在"；联系方式格式错误 → "联系方式格式不正确"；删除前确认。

**2.4.3 政府-目标分解 /gov/assess/goal**

通用：
- 顶部"报告年度" Tab：仅展示存在目标分解任务（GOAL_TASK_TYPES）的年份；年份角标"本期"=2026；空态 → AssessEmptyState（带「前往任务管理」按钮）。
- 倒计时徽标：取该年首个进行中任务的 endDate。

市级视图（city_admin）：
- 标题"全市重点单位碳排放目标分解"；二级 Tab：① 区下属单位碳排放目标分解；② "百家""千家"通信业企业碳排放目标分解。
- ① 字段（DistrictListTable）：序号、区名称、企业数、状态（未开始/进行中/已完成）、操作 → /gov/assess/goal/district/:districtId。
- ② 字段（BqGoalTable，市级只读）：区名称、信用代码、企业名称、2025 总量(吨CO₂)、2025 单位产值碳排放、推荐值、总量目标(万吨CO₂)、强度目标值、强度指标、强度单位、备注、修改徽标。
- 统计卡：参与区数 / 已完成 / 进行中 / 企业总数；百千 5 卡：企业总数 / 已完成 / 已修改 / 总量目标 / 平均强度。
- 筛选（百千）：企业名称或信用代码、是否已修改（全部/已修改/未修改）、重置；导出 Excel。

区级视图（district_admin）：
- 标题"区下属单位碳排放目标分解"。
- 字段同 ① 但展示本区下属企业，可在线编辑（CarbonGoalTable）：行内修改 total2026、intensity2026 等；保存修改产生 ChangeRecord {field, oldValue, newValue, remark, by, at}。
- 顶部操作：盖章证明（上传/预览/下载/删除/重新上传）、导出。

**2.4.4 区下属企业目标分解详情 /gov/assess/goal/district/:districtId**（市级）
- 面包屑：目标分解 / 区下属单位碳排放目标分解 / {区} 下属企业碳排放目标分解；返回列表按钮。
- 5 张统计卡 + 筛选区（关键字、已修改/未修改、重置、导出）+ CarbonGoalTable（市级 city-view，可行内调整目标值，调整即记录 ChangeRecord，by="节能中心 · {当前用户}"）。
- 异常：演示数据仅含青浦区，其它区显示占位"演示数据仅包含青浦区，其它区显示样式相同"。

**2.4.5 政府-双控考核 /gov/assess/dual**

市级视图：
- 标题"重点单位能耗双控考核结果"；二级 Tab：① 区下属单位能耗目标考核；② "百家""千家"通信业企业能耗考核。
- ① 字段：序号、区名称、企业数、状态（待考核/考核中/完成考核）、考核时间、盖章证明（已上传徽标）、操作 → /gov/assess/dual/district/:districtId。
- ② 字段（BqEntAssessTable）：序号、企业名称、年份、企业自评分、政府考评分/总分、状态、考评报告（上传/下载）、操作 → /gov/assess/dual/bq/:entId。
- 统计卡（区下属 4 张）：参与区数/完成考核/考核中/待考核；百千 5 张：企业总数/已完成/考核中/待考核/已上传报告。

区级视图：
- 字段（DistrictAssessTable）：企业名称、信用代码、能耗总量目标/实际/扣除绿电后值、强度指标/单位/目标/实际/扣除绿电后、是否达标（系统自动）、考核结果（手动选 完成/未完成）、备注。
- 4 张统计卡：企业总数/完成/未完成/达标率。
- 顶部操作：上传/预览/删除/重新上传 盖章证明、导出、保存、提交、退回。
- 状态机：待考核 → （上传盖章 + 提交）→ 完成考核（锁定，提交后表格切到只读 city-view 模式）；退回回到 待考核。
- 提示文案：「考核结果由系统自动判定，如有异议请联系区级负责人」「考核说明：总量与强度均完成视为完成；任 1 项未完成视为未完成；双控未完成但结果为完成需在备注说明」。

**2.4.6 区下属企业能耗考核详情 /gov/assess/dual/district/:districtId**（市级）
- 面包屑 + 返回；4 张统计卡（仅青浦区演示）；操作：下载盖章证明（如有）、导出、保存/提交/退回。
- 表格 DistrictAssessTable mode=city-edit：市级可调整考核结果与备注；提交后状态写入 assessStatusStore。

**2.4.7 百千通信业企业评分详情 /gov/assess/dual/bq/:entId**（市级）
- 顶部工具栏：返回、标题"企业节能"双控"责任评价考核"、企业自评附件下载、导出、下载考评报告（如已上传）、上传/重新上传考评报告（.pdf/.zip/.doc/.docx）、退回重填、保存。
- 企业基础信息卡：企业名称、考核年份、企业自评分(总分)、政府考评分(总分)、考核状态（PassBadge）。
- 分类切换 Tab：全部 + N 个分组（如：能耗总量和强度目标 30 / 节能管理 / 节能技改 等）。
- 评分项卡片字段：编号 no、单项分值 itemScore、评分标准 criterion、企业自评 {selfScore, selfBasis, proofs[{name,url}]}、政府考评 {reviewScore (0~itemScore step 0.5)、reviewBasis（必填，回车确认/Shift+Enter 换行）}。
- 状态机：待提交（企业可编辑）→ 已提交（市级可考评）→ 已完成（双方锁定）；退回重填 → 待提交。
- 异常：未找到企业 → "未找到企业数据"；reviewScore 越界 → input 控件 max/min 限制。

**2.4.8 企业-目标分解 /ent/assess/goal**

- 标题"重点单位碳排放双控目标分解"。
- 顶部归属切换 headerExtra：区管企业 / 市管企业（按当前年是否同时存在两类任务动态显示）；切到该 scope 下无任务则自动切到有任务侧。
- 报告年度按钮组：每个年份带状态点（绿=已提交，琥珀=中心已调整，灰=未提交）；右上"本期"角标。
- 状态机：草稿 → 已提交 → （中心调整）已提交·中心已调整；提交后整个 fieldset 锁定，仅提示「提交后不可修改，如需调整请联系主管部门」。
- 操作栏：填报状态徽标、保存草稿、提交（提交前 Dialog 二次确认 + 摘要：年度/归属/企业名称；提交成功 Dialog "我知道了"）。
- 区管企业表单 EntCarbonGoalForm 字段：所属区、统一信用代码、企业名称（只读）；2025 年总量、2025 单位产值碳排放（只读）；推荐值（只读，蓝色 + 说明）；2026 总量目标、强度目标（必填，可编辑）、强度指标（datalist：单位产值/产品/增加值碳排放）、强度单位（datalist：吨CO2/万元 等）；备注。
- 市管企业表单 EntBqGoalForm 字段：所属区、信用代码、企业名称；2025 总量(吨CO₂)、单位产值碳排放(吨CO₂/万元)；推荐值；2026 总量目标(万吨CO₂)、强度目标值(吨CO₂/万元)、强度指标、强度单位；备注。
- 中心调整提示 ChangeAlert：列出最近 N 条调整（字段、旧值、新值、备注、操作人、时间）；目标输入框高亮 amber 边框。

**2.4.9 企业-双控考核 /ent/assess/dual**

- 标题"重点单位能耗双控考核结果"。
- 区管企业（district 类型）：
  - 报告年度按钮组（仅有任务的年份）+ 倒计时徽标 + 右上锁提示「考核结果由系统自动判定，如有异议请联系区级负责人」。
  - 企业基础信息：所属区、信用代码、企业名称、考核状态（已达标/未达标徽标）。
  - 能耗总量目标完成情况卡：目标值、实际值、扣除绿电绿证后值（系统自动）、是否达标（PassBadge）。
  - 能耗强度目标完成情况卡：强度指标、单位、目标值、实际值、扣除绿电绿证后值、是否达标。
  - 双控考核结论卡：考核结果（区级管理员填写，企业只读）、备注；底部说明文字 ×2。
  - 异常：当年无考核结果 → 信息框「{年}年度暂无考核结果，该年度尚未完成数据归集或暂未开展考核」。
- 市管企业（city 类型）：直接调用 EntAssessDualBqBody 组件，含百千自评流程：
  - 工具栏：下载证明材料、下载考评报告（已有）、导出、保存、提交、退回；自评/考评分汇总。
  - 企业基础信息卡 + Tab 切换分组 + 评分项卡片：编辑模式下可改 selfScore（0~itemScore step 0.5）、selfBasis、上传/删除证明材料；政府考评只读。
  - 状态机：待提交（编辑）→ 已提交（可退回）→ 已完成/考核中（锁定，提示「政府已完成考评，如需修改请联系主管部门」）。

#### 2.5 通用规则

| 维度 | 规则 |
|---|---|
| 年份 Tab | 仅展示存在非"已归档"任务的年份；当前年(2026)默认选中 + "本期"角标；切换 scope/年份时若该年无任务展示空态。 |
| 倒计时徽标 | 取该年首个"进行中"任务（无则取"未开始"）的 endDate；显示剩余天数。 |
| 空态 AssessEmptyState | 三种：① 完全无任务 + 「前往任务管理」② 当前年无任务（同按钮）③ 区级"市级未下发任务"无按钮。 |
| 盖章证明（区级） | .pdf/.jpg/.png；区级未提交可上传/删除/替换；提交后仅可下载/预览。 |
| 修改记录 ChangeRecord | 区级/市级修改企业目标值即记录；展示在 ChangeAlert + 字段右侧 ChangeBadge；企业端字段高亮 amber 描边。 |
| 自动判定 | 实际值（扣除绿电绿证后）≤ 目标值 → 达标；总量与强度均达标 → 完成。 |
| 导出 | 列表 / 详情均提供 Excel 导出按钮（一期 toast 演示）。 |
| Toast | 统一 sonner，操作成功/失败提示。 |

#### 2.6 状态机汇总

```text
任务：未开始 → 进行中 → 已结束 → 已归档
区级双控：待考核 →（上传盖章+提交）→ 完成考核 →（退回）→ 待考核
区级目标分解：未开始 → 进行中 → 已完成
百千企业（自评+考评）：待提交 →（企业提交）→ 已提交 →（市级保存评分）→ 考核中 →（市级完成）→ 已完成；任意阶段可"退回重填"→ 待提交
企业目标填报：草稿 →（企业提交）→ 已提交 →（中心调整）→ 已提交·中心已调整（不可逆）
```
