# 在每个一级模块前增加"功能架构线框图"

## 目标
在 `/mnt/documents/建设内容补充_v1.docx` 现有 5 个一级模块（核心平台、政府管理智能体、企业服务智能体、绿色制造智能体、场景招商智能体）的"功能描述"前，各插入一张**简洁的功能架构线框图**，层级到子模块为止（不下钻到具体功能点），帮助阅读者一眼看清"模块 → 子模块"的组成关系。

## 线框图样式（统一规范）

- 形式：黑白线框 + 浅灰填充，纯几何方块组成的层级结构图（树状/横向分组）
- 层级：仅 2 层
  - 顶层：1 个一级模块名称框（深色填充 + 白字加粗）
  - 第二层：N 个子模块名称框（浅灰填充 + 黑字）+ 中线/直角连接线
- 风格：方正、对称、留白充足，与正文宋体保持一致；不使用图标、配色，避免花哨
- 尺寸：宽度 ≈ 正文宽度（约 15 cm），高度按子模块数自适应（120–180px）
- 位置：紧跟在每个一级模块标题（H2）之后、首段正文之前；居中排列
- 图题（caption）：例如"图 1 AI+能碳数智空间核心平台功能架构"，五号宋体居中

## 五张图的内容

1. **核心平台**：建设空间核心平台 / 构建可信底座
2. **政府管理智能体**：双控考核 / 监察管理 / 数据确权
3. **企业服务智能体**：数据治理 / 项目服务 / CBAM 应对 / 绿色金融 / 前沿咨询
4. **绿色制造智能体**：绿色评价 / 绿色改造 / 绿色运维
5. **场景招商智能体**：需求汇集 / 技术供给 / 供需撮合 / 宣传培训

## 技术实现

- 用 `matplotlib` 绘制 5 张线框图，导出为 PNG（DPI 200，背景透明/白）保存到 `/tmp/`
- 设置中文字体（fallback 至系统中文字体或下载 Noto Sans CJK），避免方块乱码
- 在 `gen_doc.py` 基础上新增 `add_diagram(path, caption)` 函数，使用 `python-docx` 的 `add_picture` 在每个 H2 后插入图片 + 图题
- 重新生成 `/mnt/documents/建设内容补充_v2.docx`
- QA：转 PDF 抽帧人工核查图示是否清晰、未溢出、未与文字重叠

## 输出
- 新文件：`/mnt/documents/建设内容补充_v2.docx`（保留 v1，便于对比）
- 字数与正文不变，仅新增 5 张线框图与图题
