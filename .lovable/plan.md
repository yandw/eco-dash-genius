# 生成"四、(一) 建设内容"补充 Word 文档（7000–8000 字）

## 目标
基于 xlsx 中"1+4+N"功能模块规划，生成一份独立 Word 文档，仅包含"建设内容"章节的补充正文，篇幅控制在 **7000–8000 字**之间，可直接插入建设方案 V1 的"四、项目建设内容和创新特色 / (一) 建设内容"。

## 文档结构（H1 / H2 / H3 三级）

**(一) 建设内容**（开篇总述约 400 字：阐述"1+4+N"总体架构、整体设计思路与各模块协同关系）

### 1. AI+能碳数智空间核心平台（约 1200 字）
- 1.1 建设空间核心平台
  - 全景监测：实时展示数据汇聚总量、强度等核心指标及五级碳账户建设
  - 生态普惠：为各类主体提供无感监管、智能核算等智能体服务
  - 数据汇聚：形成覆盖全产业链的数据资源池
  - 语料构建：梳理合规语料，构建高质量能碳语料库
- 1.2 构建可信底座
  - 数据规整：覆盖数据全生命周期的质量管控与多源交叉验证
  - 数据安全：数据可追溯不可篡改、可用不可见
- 小结·建设目标：作为"1+4+N"底座，为四大智能体提供统一数据、算力与语料支撑

### 2. 政府管理智能体（约 1700 字）
- 2.1 双控考核（指标分解 / 考核评价 / 目标拟合 / 分析预警）
  - 建设目标：碳排放双控考核覆盖 600 家以上（100%）
- 2.2 监察管理（报告审核 / 能效对标 / 能耗限额核查 / 产品碳强度监测）
  - 建设目标：企业碳排放核算覆盖规上工业 90%（超 9000 万吨 CO₂）；无感监察 500 家/年、减少现场监察 1000 人次/年；重点产品单耗 50% 以上达国标先进值
- 2.3 数据确权（交叉验证 / 证书确认 / 工作流程）
  - 建设目标：能碳数据确权 1000 家

### 3. 企业服务智能体（约 1800 字）
- 3.1 数据治理（数据清洗 / 数据流通）
- 3.2 项目服务（"两高"项目跟踪 / 项目全周期支持 / 免申即享）
  - 建设目标：节能降碳项目 1000 个；工业碳减排 100 万吨/年；通信业碳减排 5 万吨/年
- 3.3 CBAM 应对（动态解析 / 合规申报 / 安全管控 / 成本优化）
  - 建设目标：产品碳强度核算 1000 个以上
- 3.4 绿色金融（画像生成 / 金融对接）
  - 建设目标：绿电绿证交易超 50 亿 KWh/年
- 3.5 前沿咨询（信息聚合 / 智能推送）

### 4. 绿色制造智能体（约 1300 字）
- 4.1 绿色评价（对标评估 / 报告生成）
- 4.2 绿色改造（能效诊断 / 方案匹配）
- 4.3 绿色运维（数据跟踪 / 系统管理）
- 建设目标：绿色制造企业 1000 家以上、产值 3 万亿元以上

### 5. 场景招商智能体（约 900 字）
- 5.1 需求汇集 / 5.2 技术供给 / 5.3 供需撮合 / 5.4 宣传培训
- 建设目标：绿色降碳应用场景 100 个/年；绿色技术解决方案 100 个/年

### 6. 协同与小结（约 300 字）
"1 个底座 + 4 个智能体 + N 项应用"协同闭环简述

## 写作要求
- 每个子模块用 1–3 段（每段 80–180 字）阐述"做什么 / 怎么做 / 解决什么问题 / 形成什么能力"
- 每个一级模块末尾用"建设目标"小段集中列示对应"十五五"量化指标
- 行文风格与方案 V1 一致（书面、政务、技术中性）
- 全文 7000–8000 字（不含目录、标题），生成后用 `python -c "len(...)"` 校验字数，未达标自动补写

## 输出
- 路径：`/mnt/documents/建设内容补充_v1.docx`
- 中文字体：宋体（正文 12pt / 小四，行距 1.5）；标题加粗，H1 二号、H2 三号、H3 四号
- 仅含补充章节，可直接复制粘贴

## 技术实现
1. 用 `lovable_ai.py` 调用模型按章节分段生成正文（控制每节字数），减少手写工作量并保证一致风格
2. 用 `docx` (npm) 组装最终 .docx，覆盖 Heading 样式、设置中文字体
3. 校验：字数统计 + LibreOffice 转 PDF + pdftoppm 抽帧人工 QA
