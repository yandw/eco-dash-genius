## 目标

政府侧节能档案管理界面当前筛选区只支持「关键字 + 区 + 行业 + 状态」四项，而图1-3 中实际需要的筛选维度极多（对口人按科室分组、自定义企业管理标识/运行状态/历年重点/产业、主管区政府、上级主管部门按区县/集团分组、万家、行业、能耗级别）。要在不破坏列表布局的前提下让政府人员"快速选→一眼看→随时清"。

## 整体方案

保留页面顶部的"快搜行"（关键字 + 本期状态 Tab + 重置 + 导出），把所有多维标签筛选收纳到一个**右侧抽屉式高级筛选面板**，并通过「已选条件 chips 行」实时回显，整体节奏：

```text
┌─────────────────────────────────────────────────────────────┐
│ 关键字搜索 ────────────  本期状态 Tabs  [筛选 N] [重置] [导出]│
│ 已选条件: [区:浦东✕] [行业:化工✕] [对口人:吴梅✕] [清空全部]   │
└─────────────────────────────────────────────────────────────┘
┌────────── 抽屉（点击「筛选」打开，右侧滑入，宽 520px） ─────┐
│ 高级筛选          已选 N 项                          ✕      │
│ ─────────────────────────────────────────────────────────── │
│ 内嵌左侧分类导航 + 右侧标签复选区（可滚动）                 │
│  · 对口人      · 主管区政府                                  │
│  · 自定义标签  · 上级主管部门                                │
│  · 万家        · 行业                                        │
│  · 能耗级别                                                  │
│ ─────────────────────────────────────────────────────────── │
│              [重置]                  [取消] [应用筛选]      │
└─────────────────────────────────────────────────────────────┘
```

## 详细设计

### 1. 顶部"快搜行"保留并精简
**文件**：`src/pages/Archives.tsx`
- 关键字输入、所属区下拉、行业下拉合并到抽屉里，顶部仅保留：关键字搜索框 + 本期状态 Tabs + 「高级筛选」按钮 + 重置 + 导出。
- 「高级筛选」按钮显示当前未应用关键字以外的已选数量徽标，例如 `筛选 · 5`。

### 2. 已选条件 Chips 行
- 紧贴筛选行下方新增一条 chips 区，列出所有已选标签（带分组前缀，如 `对口人: 吴梅`）；每个 chip 右侧 `✕` 单独移除；末尾 `清空全部` 链接按钮。
- 当无任何高级筛选时整行隐藏，不占空间。

### 3. 高级筛选抽屉
**新增文件**：`src/components/archives/ArchiveFilterDrawer.tsx`
- 基于现有 `Sheet` 组件，从右侧滑入，宽度 `w-[520px]`，整体可滚动。
- 顶部：标题「高级筛选」+ 当前已选合计计数。
- 主体采用**左侧分类锚点 + 右侧分组卡片**两栏布局：
  - 左栏纵向 Tab：`对口人 / 自定义标签 / 主管区政府 / 上级主管部门 / 万家 / 行业 / 能耗级别`，点击平滑滚动到对应分组。
  - 右栏每个分组卡片包含：
    - 标题 + 「全选 / 取消全选」轻量链接按钮（沿用截图交互）。
    - 子分组（如对口人按"节能监察科 / 能效管理科 / 能碳监测科 / 宣教推广科"分组；上级主管部门按"区县 / 集团"分组；自定义标签按"企业管理标识 / 企业运行状态 / 历年重点 / 三大先导产业 / 六大重点产业 / 主管区政府"分组）。
    - 标签使用**可点击的 toggle chip** 替代原始 checkbox（图3 中的 checkbox 视觉信息密度低、命中区域小），未选灰描边、选中实心 primary 浅底 + primary 边框 + ✓。
- 底部固定操作栏：左侧「重置」清空所有维度；右侧「取消」「应用筛选」。仅在点击应用后才同步到列表，避免抽屉操作直接抖动列表。

### 4. 标签数据源
**新增文件**：`src/mocks/archiveFilters.ts`
- 集中维护图1-3 的所有分组与标签数据，数据结构示例：
  ```ts
  type FilterGroup = {
    key: string;
    label: string;
    subGroups: { label: string; options: string[] }[];
  };
  ```
- 所有维度（对口人/自定义/主管区/上级/万家/行业/能耗级别）按截图原文录入，不增不减。

### 5. 状态管理与列表过滤
**文件**：`src/pages/Archives.tsx`
- 维护一个 `advancedFilters: Record<groupKey, string[]>` 状态。
- `filtered` 计算时合并：
  - 关键字、本期状态（顶栏即时生效）；
  - 所属区域 ← 主管区政府 / 上级主管部门-区县（取并集，命中即匹配）；
  - 行业 ← 行业组（命中即匹配）；
  - 其他维度（对口人、自定义、万家、能耗级别）由于 mock 数据未持有这些字段，匹配函数预留「全部命中视为匹配」语义，并在 chips 上仍正常展示选择状态，方便后续接真数据。
- 「重置」按钮一次清空关键字 + 状态 + 高级筛选。

### 6. 视觉与可用性
- Toggle chip：`px-2.5 py-1 rounded-md text-xs border transition-colors`，未选 `border-border text-foreground/80 hover:border-primary/40`；选中 `bg-accent text-accent-foreground border-primary`。
- 抽屉内分组卡片：`rounded-lg border border-border/70 p-4 space-y-3`，与现有 `panel` 风格一致。
- 全部颜色走 design tokens（`primary / accent / border / muted-foreground / destructive`），不写硬编码颜色。
- 抽屉内可加一个**全局关键字搜索框**（顶部 sticky），输入即过滤所有可见标签，处理"对口人特别多"的体感。

## 影响面
- 政府侧节能档案列表页 `/gov/archives`。
- 新增筛选数据 mock 与抽屉组件，不影响其它页面。
